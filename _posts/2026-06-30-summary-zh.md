## [LocalAI: 无需 GPU 即可运行任意模型的开源 AI 引擎](https://github.com/mudler/LocalAI) ⭐️ 9.0/10

LocalAI，一个开源 AI 引擎，已发布，可在任何硬件上运行大语言模型、视觉、语音、图像和视频模型，无需 GPU。它采用可组合架构，后端如 llama.cpp 和 vLLM 按需拉取。 这消除了 GPU 的壁垒，使个人和组织能够在消费级硬件或仅 CPU 服务器上运行强大的 AI 模型，从而实现了 AI 访问的民主化。同时，模型在本地运行，无需将数据传输到外部服务，确保了数据隐私。 LocalAI 设计为一个小型核心，没有捆绑依赖；后端如 llama.cpp、whisper.cpp 和 stable-diffusion 被容器化，仅在需要时拉取。它提供与 OpenAI、Anthropic 和 ElevenLabs 的即插即用 API 兼容性，并支持多用户功能，如 API 密钥认证和基于角色的访问。

## [WATaBoy：将 Game Boy 指令 JIT 编译为 WebAssembly，性能超越原生解释器](https://humphri.es/blog/WATaBoy/) ⭐️ 8.0/10

一个名为 WATaBoy 的项目展示了即时编译（JIT）技术，将 Game Boy 指令翻译成 WebAssembly，其性能超过了本机解释器。 该方法表明，利用 WebAssembly 在浏览器中的 JIT 能力，可以在模拟领域超越传统的本机解释器，为在 iOS 等受限平台（JIT 仅允许在浏览器中使用）上实现高性能模拟开辟了新途径。 该项目表明，尽管 WebAssembly 相比本地代码有约 20%的开销，但由于解释器的开销可高达 1000%，JIT 方法仍能胜出。不同浏览器性能存在差异，Firefox 比 Chrome/Safari 慢约 25%。

hackernews · energeticbark · 6月29日 15:02 · [社区讨论](https://news.ycombinator.com/item?id=48720190)

**背景**: Game Boy 模拟通常通过逐一解释每条指令来运行，速度较慢。即时编译（JIT）在运行时将频繁执行的代码翻译成本地机器码以加速。WebAssembly 是一种低级二进制格式，可在现代浏览器中以接近本机的速度运行，并支持浏览器内的 JIT 编译。通过以 WebAssembly 为目标，该项目可以在任何支持 WebAssembly 的平台上运行，包括 iOS——传统 JIT 在 iOS 上被禁止，但浏览器例外。

## [CUDA 内核启动内部机制详解](https://fergusfinn.com/blog/what-happens-when-you-run-a-gpu-kernel/) ⭐️ 8.0/10

这篇文章填补了理解 CUDA 内核启动内部机制的关键空白，对于追求性能优化的 GPU 程序员非常有价值。它揭示了通常晦涩的驱动与硬件交接过程，帮助开发者编写更高效的代码。 启动过程使用内存映射的门铃寄存器向 GPU 发送信号，QMD 被传递给计算工作分发器。默认流隐式处理同步，而显式流则允许并行命令执行。

hackernews · mezark · 6月29日 13:11 · [社区讨论](https://news.ycombinator.com/item?id=48718863)

**背景**: CUDA 是 NVIDIA 的并行计算平台和编程模型，用于 GPU 加速。当 CUDA 内核启动时，CPU 准备一个命令描述符（QMD）并写入 GPU 内存，然后触发门铃寄存器通知 GPU。GPU 的硬件命令队列拾取描述符并调度在流式多处理器上执行。CUDA 流允许多个操作并发执行，并需要显式同步以保证正确性。
